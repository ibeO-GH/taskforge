# TaskForge

TaskForge is a modern task management system built across multiple frontend frameworks to explore differences in architecture, state management, and UI patterns.

## 🚀 Implementations

- ⚛️ React → https://github.com/ibeO-GH/taskforge-react
- ▲ Next.js → https://github.com/ibeO-GH/taskforge-next
- 🟢 Vue → https://github.com/ibeO-GH/taskforge-vue

## ✨ Features

- Create, edit, delete tasks
- Search and filtering
- Task detail view
- Persistent storage (IndexedDB)
- Clean, modern UI
- Consistent UX across frameworks

## 🧠 Why I Built This

I wanted to go beyond learning frameworks individually and instead understand:

- How the same product is structured across different ecosystems
- Differences in state management (React vs Vue)
- Routing approaches (Next.js vs Vue Router)
- Reusability and scalability patterns

## 🛠️ Tech Stack

- React (Vite)
- Next.js (App Router)
- Vue (Composition API + Pinia)
- TypeScript
- Tailwind CSS
- Dexie (IndexedDB)

## 🎯 Key Takeaway

This project demonstrates that strong frontend engineering is not about knowing a single framework, but understanding how to build scalable systems regardless of the tools used.
